## 一、产品概述

### 1.1 产品背景

抖音生活服务近年来高速增长，平台上活跃着大量餐饮、酒旅、休闲娱乐等中小商家。这些商家普遍面临同一个痛点：**有内容营销需求，但没有专业内容团队**。

- 雇人贵：专职内容运营月薪 8,000–15,000 元，中小商家难以承担
- 外包慢：外包制作周期长（3–7 天），无法满足即时营销节点需求
- 门槛高：商家自己写文案、拍视频、配图，能力参差不齐，效果不稳定

与此同时，大语言模型（LLM）和 AI 生图/视频技术已具备商业化条件，**"一个主题输入，一站式内容输出"** 的产品模式在技术上可行，在需求上也真实存在。

### 1.2 产品定位

| 维度 | 描述 |
|------|------|
| 产品名称 | 抖店内容工坊 |
| 产品定位 | 面向抖音生活服务商家的 **SaaS 一站式内容创作平台** |
| 核心价值主张 | 商家输入一个营销主题，平台智能生成**文案 + 图片 + 视频 + 直播物料** |
| 差异化 | 垂直于抖音生活服务场景，开箱即用，无需专业技能 |
| 商业模式 | SaaS 订阅制（免费基础功能 + 付费高级功能） |

### 1.3 目标用户

| 用户画像 | 描述 |
|----------|------|
| 典型用户 | 餐饮（火锅、烧烤、奶茶等）、酒旅（民宿、景区）、休闲娱乐（剧本杀、KTV、密室逃脱）等行业的中小商家 |
| 用户规模 | 目标：抖音生活服务商家数百万中的首批付费转化用户 |
| 核心需求 | 快速获取高质量营销内容，降低内容生产成本 |
| 痛点优先级 | ① 文案不会写 > ② 图片不会做 > ③ 视频制作难 > ④ 直播物料准备麻烦 |
| 技术能力 | 基础手机操作即可，无需设计或技术背景 |
| 决策者 | 老板本人或店面运营负责人（通常是同一个人） |

### 1.4 MVP 目标

**本轮 MVP 目标：验证核心链路，跑通"输入主题 → 生成文案"完整闭环。**

具体指标：
- 功能可用率：首页、文案创作两大模块端到端可用
- 用户任务完成率：商家完成一次文案生成的完整路径 ≤ 5 分钟
- 付费转化基准：MVP 上线后 30 天，自然流量付费转化率 > 3%（待定）

> **范围红线**：MVP 不做真正的 AI 图片生成、AI 视频生成，仅做展示页和占位符。付费功能按"引导页 → 功能页"两步走。

## 二、功能需求详述

### 2.1 首页工作台

**功能概述**：用户打开产品的第一个页面，是整个产品的入口中枢。

**功能点清单**

| 功能点 | 描述 | 备注 |
|--------|------|------|
| 顶部导航栏 | Logo + 产品名称 + 会员状态入口（免费/付费标识） | 会员状态与后端用户系统联动 |
| 营销主题输入框 | 中央醒目位置，支持输入营销主题（如"火锅店周年庆"），点击进入创作流 | 占位文案示例："例如：火锅店周年庆 / 民宿春季促销" |
| 四大模块入口卡片 | 四张卡片横向排列：文案创作 / 图片设计 / 短视频 / 直播物料 | 点击文案创作进入创作页；其余三个展示引导页 |
| 会员状态展示 | 右上角显示"免费用户"或"付费会员"标签 | 免费用户点击付费功能卡片时触发开通引导 |

**交互说明**

- 主题输入框获得焦点（focus）时，placeholder 文字淡出
- 输入内容后，底部出现"开始创作"按钮
- 点击"开始创作"或直接按回车，跳转文案创作页（主题自动带入）
- 四大卡片：文案创作卡片高亮（可用），其余三张灰色/加锁态（引导付费）
- 付费用户：四张卡片均可点击

**边界情况**

- 输入为空时点击"开始创作"：输入框抖动 + 提示"请输入营销主题"
- 超过 100 字：截断，不允许继续输入
- 未登录用户：点击任意功能入口，先跳转登录页

---

### 2.2 文案创作模块（免费基础功能）

**功能概述**：MVP 核心功能。用户选择文案类型、输入行业信息，AI 生成多版本文案，支持复制和导出。

**功能点清单**

| 功能点 | 描述 | 备注 |
|--------|------|------|
| 文案类型选择 | 四选一：脚本 / 直播话术 / 种草文 / 评论回复 | 单选，点击切换 |
| 行业选择 | 下拉选择：餐饮 / 酒旅 / 休闲娱乐 / 通用 | 必填 |
| 品类输入 | 文本输入，如"火锅"、"民宿"、"剧本杀" | 必填，示例占位符 |
| 营销主题输入 | 预填来自首页的主题，支持修改 | 必填 |
| 补充信息（可选） | 富文本框，支持输入活动力度、优惠信息、店铺特色等补充说明 | 选填，帮助 AI 生成更精准 |
| 生成按钮 | "AI 生成文案"主按钮 | 生成中显示 loading 动画 |
| AI 生成结果区 | 展示 2–3 个文案版本，版本之间可切换 | 每个版本有"使用此版本"按钮 |
| 复制功能 | 一键复制当前版本文案到剪贴板 | 显示"已复制"Toast 提示 |
| 导出功能 | 导出为 .txt 文件 | 浏览器原生下载 |
| 历史记录入口 | 展示最近 5 条创作记录，点击可重新编辑 | 非 MVP 核心，留入口但功能可简化 |

**用户流程**

```
首页 → 点击"文案创作"卡片/输入主题后"开始创作"
  → 文案创作页（类型 + 行业 + 品类 + 主题 + 补充信息）
  → 点击"AI 生成文案"
  → 加载中（1–3 秒）
  → 展示 2–3 个版本
  → 选择版本 → 复制 / 导出
  → 结束
```

**交互说明**

- 类型选择：以 Tab 切换形式，选中态高亮
- 生成中：按钮变为"生成中…" + loading spinner，不可重复点击
- 生成失败：展示错误提示"生成失败，请重试"，按钮恢复可点击
- 版本切换：卡片式切换，当前选中版本边框高亮

**边界情况**

- 未填必填项（行业/品类/主题）：对应字段标红 + "此项为必填"
- 生成超时（>10 秒）：提示"生成超时，请重试"
- AI 返回为空：提示"生成内容为空，请重试"
- 复制失败（浏览器限制）：降级为"选中文字"提示

### 2.3 图片设计展示页（付费引导页）

**功能概述**：图片设计是 MVP 的付费功能，当前阶段仅展示引导页，引导用户开通会员。

**功能点清单**

| 功能点 | 描述 | 备注 |
|--------|------|------|
| 模板展示区 | 横向滚动展示 6–12 张模板图（商品套图 / 海报 / 店铺装修图），配有简短标签 | 仅展示，无交互 |
| 标签筛选 | 筛选标签：全部 / 商品套图 / 海报 / 店铺装修 | 静态展示 |
| 功能说明文案 | 文字说明："AI 智能生成 · 多种尺寸适配 · 抖音平台优化" |  |
| 付费引导 banner | 底部固定 banner："开通会员解锁全部图片设计功能" + 跳转按钮 | 付费用户不显示 |
| 会员状态判断 | 付费用户：banner 隐藏，正常展示功能入口 |  |

**交互说明**

- 模板图片点击：弹出大图预览弹窗（付费引导弹窗）
- 开通会员按钮：跳转至会员开通页（本期接入支付渠道，后续实现）

**边界情况**

- 付费用户：banner 不展示，直接进入功能页（功能页本期用占位页代替）
- 未登录用户点击开通：跳转登录页，登录后回跳

## 三、非功能需求

### 3.1 性能需求

| 指标 | 要求 |
|------|------|
| 页面首屏加载时间 | ≤ 2 秒（3G 网络） |
| AI 文案生成时间 | ≤ 5 秒（p99 ≤ 10 秒） |
| 页面可交互时间（TTI） | ≤ 3 秒 |
| 崩溃率 | ≤ 0.1% |

### 3.2 兼容性需求

- 浏览器：Chrome 80+、Safari 14+、Firefox 75+、Edge 80+
- 移动端：iOS Safari 13+、Android Chrome 80+
- 响应式：移动端优先，支持手机端完整使用

### 3.3 安全需求

- 用户输入内容做 XSS 过滤
- AI 生成内容做违规词初步过滤（接入抖音内容安全 API，待确认）
- 登录态 Token 有效期 ≤ 7 天，支持刷新

### 3.4 监控与可观测性

- 埋点覆盖：页面PV、按钮点击、文案生成成功率、生成耗时
- 错误日志：前端 JS Error 捕获上报
- AI 调用：记录请求量、成功率、平均耗时

| ID | 用户故事 | 验收条件 |
|----|----------|----------|
| US-01 | 作为餐饮商家，我想在首页输入"火锅店周年庆"并快速生成周年庆文案，无需注册即可体验 | 未登录用户在首页输入主题后点击"开始创作"，跳转登录页；登录完成后主题自动带入文案创作页 |

## 五、验收标准（Acceptance Criteria）

### 5.1 首页工作台

| 验收项 | Given-When-Then |
|--------|-----------------|
| AC-01 | **Given** 用户打开首页，**When** 页面加载完成，**Then** 主题输入框和四张模块卡片均可见 |
| AC-02 | **Given** 用户未输入主题，**When** 点击"开始创作"，**Then** 输入框抖动，显示"请输入营销主题"提示 |
| AC-03 | **Given** 免费用户，**When** 点击图片设计卡片，**Then** 弹出付费引导弹窗，不进入功能页 |
| AC-04 | **Given** 付费用户，**When** 点击任意卡片，**Then** 正常跳转进入对应功能页 |
| AC-05 | **Given** 主题输入超过100字，**When** 用户继续输入，**Then** 不允许输入，超长内容被截断 |

### 5.2 文案创作模块

| 验收项 | Given-When-Then |
|--------|-----------------|
| AC-06 | **Given** 用户在文案创作页，**When** 填写行业+品类+主题后点击"AI生成文案"，**Then** 按钮变为loading态，2-5秒后展示2-3个文案版本 |
| AC-07 | **Given** AI生成中，**When** 用户再次点击生成按钮，**Then** 按钮不可点击，无重复请求 |
| AC-08 | **Given** AI生成失败，**When** 返回错误，**Then** 显示"生成失败，请重试"，按钮恢复可点击 |
| AC-09 | **Given** 文案结果展示后，**When** 用户点击"复制"按钮，**Then** 剪贴板内容为当前版本文案，显示"已复制"Toast |
| AC-10 | **Given** 文案结果展示后，**When** 用户点击"导出"按钮，**Then** 浏览器下载 .txt 文件，文件名含营销主题关键词 |
| AC-11 | **Given** 用户未填必填项，**When** 点击生成，**Then** 对应字段高亮红色，显示"此项为必填" |
| AC-12 | **Given** 用户在文案结果页切换版本，**When** 点击另一版本，**Then** 内容区切换为新版本，之前版本内容不丢失 |

### 5.3 图片设计展示页

| 验收项 | Given-When-Then |
|--------|-----------------|
| AC-13 | **Given** 免费用户打开图片设计页，**Then** 模板展示区和付费引导banner同时可见 |
| AC-14 | **Given** 付费用户打开图片设计页，**Then** 无付费引导banner，页面内容与免费用户一致 |
| AC-15 | **Given** 用户点击任意模板图，**Then** 弹出大图预览（免费用户）或进入编辑页（付费用户） |

### 5.4 短视频流程页

| 验收项 | Given-When-Then |
|--------|-----------------|
| AC-16 | **Given** 用户打开短视频页，**Then** 流程示意图可见，步骤包含"脚本→确认→AI生成视频→下载使用" |
| AC-17 | **Given** 用户点击流程步骤，**Then** 弹出"功能即将上线"提示 |

## 六、技术方案概要

> 以下为本期 MVP 技术方案概要，详细设计由后端/前端团队补充。

### 6.1 技术栈（建议）

| 层级 | 技术选型 |
|------|----------|
| 前端框架 | React + TypeScript（支持 SSR/SSG） |
| 移动端适配 | Tailwind CSS（响应式） |
| 状态管理 | Zustand |
| AI 调用 | 后端代理 OpenAI / 字节豆包 API（文案生成本期接入） |
| 静态部署 | Vercel / 字节云 |
| 登录鉴权 | 飞书 OAuth 2.0（本期），支持扩展微信/手机号 |

### 6.2 AI 文案生成接口（本期接入）

```
POST /api/v1/content/generate
Body: {
  type: "script" | "live_script" | "review" | "comment_reply",
  industry: string,
  category: string,
  topic: string,
  extra_info?: string
}
Response: {
  success: boolean,
  versions: Array<{ id: string, content: string }>,
  error?: string
}
```

### 6.3 MVP 前后端分工

| 模块 | 前端 | 后端 |
|------|------|------|
| 首页工作台 | 静态页面 + 状态判断 | 用户登录态/会员状态接口 |
| 文案创作页 | 表单 + 调用生成接口 + 结果展示 | AI 生成接口（接入豆包/Doubao） |
| 图片展示页 | 静态展示 + 会员状态判断 | 会员状态接口 |
| 短视频页 | 静态占位页 | 无 |

## 八、风险评估

| 风险项 | 等级 | 应对措施 |
|--------|------|----------|
| AI 文案质量不达预期 | 🟡 中 | 建立人工评测集，首批上线前做人工抽检；预留 prompt 调优时间 |
| AI API 调用成本超预算 | 🟡 中 | 设置单用户每日生成次数上限；监控调用量和单价 |

## 九、附录

### 9.1 词汇表

| 术语 | 定义 |
