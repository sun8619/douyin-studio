# 抖店内容工坊 · 品牌化脚本生成方案

> 解决核心问题：**生成的短视频脚本太泛，跟品牌/门店没关系，用不了**

---

## 一、问题根源分析

### 1.1 现有脚本模板的问题

| 问题 | 具体表现 | 后果 |
|------|---------|------|
| **占位符泛滥** | 脚本中大量使用"XX火锅"、"成都玉林路"等通用词 | 用户拿到脚本还要手动替换，易出错 |
| **缺乏品牌个性** | 所有奶茶店用同一套"芝士浓郁、茶香清冽"描述 | 无法区分喜茶和蜜雪冰城 |
| **产品信息缺失** | 不提及具体招牌产品、口味特色 | 脚本没有记忆点，转化率低 |
| **情感共鸣不足** | 用户评价关键词从未被使用 | 脚本缺乏真实感，像机器写的 |
| **差异化卖点缺位** | 不知道门店有何不同 | 脚本跟竞品比没有竞争力 |

### 1.2 什么才是好的门店营销脚本？

```
好脚本 = 真实品牌信息 + 差异化记忆点 + 情感共鸣 + 精准引流
```

**四个维度缺一不可：**

1. **品牌真实信息**：招牌产品名称、价格、地址、联系方式必须准确
2. **差异化记忆点**：这个店和竞品最大的不同是什么（"首家陈皮柠檬茶"、"30年老字号"）
3. **情感共鸣词**：来自真实用户评价（"上头"、"回购"、"绝了"）比营销话术更有说服力
4. **精准引流路径**：结尾引导关注、到店、下单，必须和门店活动匹配

## 二、品牌化脚本生成方案

### 2.1 整体流程

```
用户输入品牌名 → 联网搜品牌信息 → 提取结构化品牌情报 → 注入脚本模板 → 生成3个版本
                    ↓（搜不到时）
              用同品类优秀品牌兜底
```

### 2.2 品牌信息搜索方案

#### 信息来源优先级

| 优先级 | 来源 | 可获取信息 | 接入方式 |
|--------|------|-----------|---------|
| **P0** | 本地品牌数据库 | 品牌名、招牌产品、调性、评价关键词、差异化卖点 | 内置，支持16+品类 |
| **P1** | 大众点评商家API | 评分、评价关键词、招牌菜、价格区间 | 需申请资质 |
| **P2** | 抖音开放平台 | 门店信息、用户讨论热词、热门视频 | 需申请资质 |
| **P3** | 小红书品牌数据 | UGC笔记关键词、用户真实反馈、种草文案 | 需申请资质 |

#### 本地品牌数据库覆盖

```
品牌数据库 (src/data/brand-database.ts)
├── 奶茶/柠檬茶：柠季、啊叻奶茶、霸王茶姬、喜茶、古茗
├── 火锅：海底捞、巴奴毛肚火锅
├── 烤肉：西塔老太太
├── 咖啡：瑞幸咖啡
└── 烘焙：泸溪河
```

#### 搜索逻辑（三层兜底）

```
Step 1: 本地数据库精确匹配（品牌名含关键词）
   ↓ 找不到
Step 2: 联网搜索（大众点评/抖音/小红书API）
   ↓ 找不到
Step 3: 用同品类评分最高的品牌作为参考
```

**示例：用户输入"廿四黑陈皮柠檬茶"**

- Step 1：本地库搜"廿四黑" → ❌ 无
- Step 2：联网搜 → ❌ 搜不到（知名度不足）
- Step 3：用"柠季"（柠檬茶品类评分4.7分）作为参考品牌
  - 脚本中自动填充：陈皮柠檬茶的酸甜比例描述、香水柠檬特点
  - 标注"📋 品类参考：基于柠季优秀案例生成"

### 2.3 品牌情报提取字段

```typescript
interface BrandInfo {
  name: string;           // "柠季·手打柠檬茶"
  tagline: string;         // "手打柠檬茶，只用好柠檬"
  vibe: '小清新' | '潮流' | '治愈' | ...;  // 品牌调性
  signature: [             // 招牌产品
    { name: '招牌柠檬茶'; tag: '招牌'; description: '广东香水柠檬现切现捣' }
  ];
  reviewKeywords: string[];  // ['清爽', '解腻', '好喝不贵']
  emotionalKeywords: string[]; // ['回购', '上头', '夏天必备']
  differentiators: string[];  // ['广东香水柠檬', '现切现捣']
  priceRange: string;      // "12-18元"
  avgRating: string;       // "4.7分"
  promotion: string;       // "第二杯半价"
}
```

## 四、品类脚本优化模板（Markdown）

### 4.1 模板说明

每个行业的脚本模板包含：
- **3-5个开场钩子变体**（每次随机选一个，避免重复）
- **3个主体段落**（产品特写 + 场景共鸣 + 节奏剪辑）
- **2-4个结尾引流变体**（根据营销活动自动匹配）

## 六、实施说明

### 6.1 文件结构

```
src/
├── app/api/
│   ├── brand/search/route.ts   ← 品牌搜索API（联网+本地数据库）
│   └── text/generate/route.ts  ← 品牌化文案生成API
├── data/
│   └── brand-database.ts       ← 品牌情报数据库（16+品类）
└── app/(app)/create/text/page.tsx  ← 前端UI（已集成品牌搜索）
```

### 6.2 联网搜索接入建议

当前版本使用**本地品牌数据库兜底**，实际部署时建议按优先级接入：

1. **大众点评商家搜索API**（推荐首批接入）
   - 可根据城市+品类搜索附近门店
   - 返回评分、评价摘要、招牌菜

2. **抖音开放平台门店API**（第二批）
   - 获取门店POI信息
   - 获取用户UGC视频/评论热词

3. **小红书品牌数据API**（第三批）
   - 获取UGC种草内容关键词
   - 获取真实用户语言风格

### 6.3 常见问题

**Q: 品牌库中没有的门店怎么办？**
A: 自动使用同品类评分最高的参考品牌生成脚本，并在版本标签中标注"品类参考"。

**Q: 如何提高脚本的品牌定制质量？**
A: 建议商家在大众点评/抖音上完善门店信息（招牌菜、地址、标签），系统会自动抓取。

**Q: 脚本生成失败时怎么办？**
A: 系统会自动降级到通用模板，确保总有结果输出。
