# 文案创作页面 UI 优化方案

> **抖店内容工坊 · 文案创作模块 V2.0**
> 设计师：UI 设计师
> 日期：2026-05-15
> 版本：V1.0（初稿）

---

## 一、设计背景与目标

### 1.1 问题分析

根据用户反馈，现有文案创作模块存在以下核心问题：

| 问题 | 影响 |
|------|------|
| 输入信息太少 | 生成的脚本与实际门店/品牌信息脱节 |
| 缺少门店名称 | AI 无法准确关联品牌调性，脚本个性化程度低 |
| 品类+描述体验不佳 | 现有下拉+文本框组合不够直观，信息密度低 |
| 模块间无串联 | 脚本生成后无法直接流转到视频/图片生成，用户操作断链 |

### 1.2 设计目标

1. **信息丰富度提升**：新增门店名称、品牌简介、营销目标等关键输入字段
2. **输入体验优化**：采用卡片式选择+自由描述组合，降低填写门槛
3. **工作流串联**：生成结果卡片底部增加「去生成视频/图片」快捷入口
4. **视觉一致性**：保持现有抖音红+渐变品牌色，遵循 8px 网格系统

### 4.3 工作流串联按钮（核心新增 ⭐）

**设计规格：**

| 属性 | 视频按钮 | 图片按钮 |
|------|----------|----------|
| **背景色** | `bg-gradient-to-r from-red-500 to-orange-400` | `bg-gradient-to-r from-purple-500 to-pink-400` |
| **图标** | 🎬 | 🖼️ |
| **文字** | ✅ 确认脚本，去生成视频 → | 🖼️ 确认脚本，去生成图片 → |
| **hover** | `from-red-600 to-orange-500` + scale(1.02) | `from-purple-600 to-pink-500` + scale(1.02) |
| **圆角** | 10px | 10px |
| **高度** | 44px | 44px |
| **动画** | 左侧箭头 pulse 效果 | 左侧图标 pulse 效果 |

**交互逻辑：**

```typescript
interface WorkflowAction {
  type: 'video' | 'image';
  targetPath: '/create/video' | '/create/image';
  scriptId: string;
  scriptContent: string;
  storeName: string;
  marketingGoal: string[];
}

// 点击后的行为：
1. 保存当前脚本到 Context/Store（storeName + scriptContent + marketingGoal）
2. 跳转路由到目标页面（/create/video 或 /create/image）
3. 目标页面自动填充：
   - 视频：脚本内容 + 门店名称
   - 图片：脚本摘要 + 门店名称
4. 显示 toast：「脚本已同步，正在跳转...」
```

**视觉层次：**
- 按钮位于结果卡片底部，独立区块
- 上方用 `separator` 分隔
- 两个按钮等宽并排（间距 12px）
- 按钮内文字左对齐，箭头/图标右对齐

### 5.4 交互规范

**动画规格：**

| 交互 | 动画类型 | 时长 | 缓动函数 |
|------|----------|------|----------|
| 按钮 hover | scale(1.02) + 阴影加深 | 200ms | ease-out |
| 串联按钮箭头 | pulse（opacity 0.7→1） | 1500ms | ease-in-out（循环） |
| 结果卡片出现 | fadeIn + slideUp | 300ms | ease-out |
| 加载骨架 | shimmer 动画 | 2000ms | linear |
| Tag 选中 | border-color + bg-color 变化 | 150ms | ease |

**错误处理：**

| 场景 | 处理方式 |
|------|----------|
| 必填字段为空 | 输入框边框变红 + 下方错误提示 |
| 生成失败 | Toast 提示 + 重试按钮 |
| 复制成功 | Toast：「已复制到剪贴板」 |

**空状态：**

| 场景 | 设计 |
|------|------|
| 未填写信息 | 预览区显示引导卡片 + 填写提示 |
| 无历史记录 | 暂无创作记录，开始你的第一次创作 |
